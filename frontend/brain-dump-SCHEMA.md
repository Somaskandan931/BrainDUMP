# Database Schema — Milestone 2

SQLite via SQLAlchemy 2.0 declarative models (`Mapped` / `mapped_column`).
Engine, session factory, and `init_db()` live in `backend/database.py`.

## Entity-relationship overview

```
Project (1) ───< (many) Task ───< (many) Subtask
                         │  │
                         │  └───< (many) WorkSession >─── (0/1) CalendarEvent
                         │
                         └───< (many) Prediction

CalendarEvent (0/1) ───< (many) WorkSession   [optional link, see below]

ProductivityMetric   — standalone daily rollup, not FK'd to anything
Setting              — standalone key/value store
```

## Tables

| Table | Purpose | Key columns |
|---|---|---|
| `projects` | Groups related tasks | `name`, `status`, `goal_text` (set if generated by the Goal Engine) |
| `tasks` | Central work item | `project_id` (nullable — standalone tasks allowed), `status`, `importance`, `energy_requirement`, `deadline`, `estimated_hours`/`actual_hours`, `confidence_score`, `priority_score`, `context_switch_cost` |
| `subtasks` | AI Task Breakdown output | `task_id`, `title`, `status`, own `estimated_hours`/`actual_hours` |
| `work_sessions` | Ground-truth timer/focus sessions | `task_id`, `calendar_event_id` (nullable), `start_time`/`end_time`, `duration_minutes` vs `predicted_duration_minutes` |
| `calendar_events` | Local cache of Google Calendar + Brain-Dump-created blocks | `task_id` (nullable), `google_event_id` (unique, for sync reconciliation), `source`, `sync_status` |
| `predictions` | Estimator/Priority training log | `task_id`, `category`, `predicted_hours` vs `actual_hours`, `error_pct`, `predicted_priority_score` |
| `productivity_metrics` | Daily rollup for Analytics/Weekly Review | `date` (unique), `hours_worked`, `completion_rate`, `most_productive_hour`, `estimation_error_pct` |
| `settings` | Generic key/value preferences | `key` (unique), `value` (JSON-encoded text) |

All tables inherit `created_at` / `updated_at` (UTC, auto-managed) from
`TimestampMixin` in `models/mixins.py`.

## Design notes

- **`Task.project_id` is nullable.** A brain dump like "Gym tomorrow" is a
  valid task with no project — matching the PRD's brain-dump examples.
  Every other FK to `tasks.id` cascades on delete, so removing a task
  cleans up its subtasks, sessions, calendar events, and predictions.
- **`Subtask` is a separate table, not a self-referential `Task`.**
  Subtasks don't carry a priority score or energy requirement — they
  inherit urgency from the parent task, and keeping them a distinct,
  simpler model matches how the AI Task Breakdown agent will emit them
  (Milestone 4).
- **`WorkSession` is the ML ground truth.** `predicted_duration_minutes`
  is set at scheduling time; `duration_minutes` is filled when the
  session ends. `ml/estimator.py` (Milestone 8) trains directly on this
  pair rather than on `Task.estimated_hours`/`actual_hours`, which are
  denormalized convenience fields updated from sessions.
- **`CalendarEvent` serves two roles**: a local cache of pulled Google
  Calendar events (`source=GOOGLE`, for fast free/busy lookups) and
  scheduler-created work blocks pushed outward (`source=BRAIN_DUMP`,
  tracked via `sync_status`). `google_event_id` is unique and nullable
  so local-only events don't collide.
- **Enums are Python `str, enum.Enum`** (`models/enums.py`) so values
  serialize cleanly to JSON for the FastAPI layer without a custom
  encoder, while still being validated at the DB layer via
  SQLAlchemy's `Enum` type.

## Verified

Milestone 2 was validated by running `init_db()` against a throwaway
SQLite file, inserting a `Project` → `Task` → `Subtask` → `WorkSession` →
`Prediction` chain, and reading it back through the ORM relationships.
All 8 tables and foreign keys came up as designed.
